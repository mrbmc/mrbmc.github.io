One of the core theses that has guided my career, and my design principles, is that **making software is really hard**. It's even more difficult than you think, my dear reader, and you already know how hard it is.

qualifier: good software is hard. anyone can make quick brittle junk. more on this later.

There are many reasons why software is so challenging to build: diverse human experiences, systems entropy, inscrutable lexicons, CSS... If you pull the threads on most of these you unravel the problem down to a simple foundational truth: 

>Code is easier to write than it is to read.

## outline

software is hard
there is a lot of crappy software

the good software takes time, it doesn't do a lot, and it is comparatively expensive.

The reason software is hard

Building software requires many disciplines

Building anything of value requires effort. time. cycles of iteration. You have to understand the past, and current, in order to manifest the future.

reading is harder than writing.
reading is a form of critical thinking.

at least reading your own code you have a snowball's chance of understanding what it does, and how it intersects with other systems.

when we abdicate the critical thinking behind code to a machine, the code becomes unmaintainable. it becomes impossible to iterate on. The finesse of quality becomes exponentially harder when working on a black box.

the low barrier of entry of these current tools enable people to produce crummy software faster. I can't help be skeptical of the value of your AI Agent you made in 3 hours. Value requires effort?

Until AI agents can operate on themselves at our behest, we will not have the tools to control the software that we create.
